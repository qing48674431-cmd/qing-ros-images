# 安全策略

本仓库提供 RouterOS CHR 镜像与一键安装脚本。以下说明本仓库的受支持版本及漏洞报告方式。

## 受支持版本

当前对本仓库**脚本与发布流程**提供维护的版本如下（镜像与系统安全更新以 MikroTik / 上游 CHR 源为准）：

| 版本     | 支持情况 |
| -------- | -------- |
| v7.21.3  | ✅ 支持  |
| v7.20.6  | ✅ 支持  |
| 其他版本 | ❌ 不保证维护 |

RouterOS 系统本身的安全更新与漏洞修复请参考 [MikroTik 官方](https://mikrotik.com/) 及 [elseif/MikroTikPatch Releases](https://github.com/elseif/MikroTikPatch/releases)。

## 报告漏洞

如发现与本仓库**脚本、发布流程或使用方式**相关的安全问题（例如脚本逻辑缺陷、不当权限、敏感信息泄露等），请通过以下方式报告：

1. **优先**：在 GitHub 仓库中 [新建 Issue](https://github.com/qing48674431-cmd/qing-ros-images/issues/new)，标题注明「Security」或「安全」。
2. 如涉及敏感细节、不希望公开讨论，可在 Issue 中简要说明问题类型，并注明希望私下联系，维护者会视情况通过其他渠道跟进。

**说明：**

- 本仓库仅提供安装脚本与镜像分发，**RouterOS 系统漏洞**请向 MikroTik 或相关上游项目报告。
- 我们会尽快确认是否受理，并在受理后根据需要回复处理进度；是否修复、何时修复将视严重程度与维护能力而定。
- 若报告被采纳，会在修复方案就绪后（在可行范围内）在 Release 或 Issue 中说明，恕不承诺具体时间表。

感谢你帮助本仓库与使用它的用户更安全。
